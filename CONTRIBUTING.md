# Contributing

Thank you for your interest in contributing to php-deploykit!

## Reporting bugs and security issues

- Open a GitHub issue for any bugs, feature requests, or general problems.
- For security vulnerabilities, please use GitHub’s private vulnerability reporting feature (found under the “Security” tab of the repository). **Never report security vulnerabilities publicly**

## Pull request guidelines

To contribute code changes, follow the standard GitHub pull request workflow:

1. Fork the repository.
2. Create a new branch for your work.
3. Make whatever changes you need — small or large — and use commit messages that briefly describe the intent.
4. If documentation updates are required, open a second PR in the [deploykit-docs](https://github.com/nathanael-thms/deploykit-docs) repository.  
   Reference the code PR in the docs PR, and the docs PR in the code PR. You may reference the PRs in a comment

5. Open a pull request against the `main` branch.

A good PR includes:

- A clear description of the problem and the proposed solution.
- Any setup or testing steps required to verify the change.
- References to related issues, if applicable.
